Introduction

React is a popular JavaScript library for building user interfaces. Developed and maintained by Facebook, it allows you to create interactive, reusable, and modular UI components. React is known for its performance, flexibility, and the use of a virtual DOM, which optimizes updates and rendering.

Key Term

  1. Component: A core building block in React. Components are reusable, self-contained pieces of the user interface that can be created and composed to build complex applications.
  2. Virtual DOM: A lightweight representation of the actual DOM (Document Object Model). React uses the virtual DOM to improve rendering performance by minimizing the number of updates to the actual DOM.
  3. Props: Short for "properties," props are inputs passed to a component. They are used to configure and customize a component's behavior and appearance.
  4. State: State is a way to manage and store data within a component. It allows a component to maintain and track changes in its data and re-render as needed.
  5. JSX (JavaScript XML): A syntax extension for JavaScript that allows you to write HTML-like code within your JavaScript. JSX is used to define the structure and appearance of React components.
  6. npm: npm is a package manager for JavaScript that comes bundled with Node.js. It is used to manage and install packages, including React.
  7. yarn: An alternative package manager to npm, specifically designed for speed and reliability when installing packages.
  8. Import: The import statement is used to include external modules or libraries (in this case, the React library) in your JavaScript file.
  9. Render: The render method in a component is responsible for returning the JSX that defines the component's structure and appearance.
  10. Prop Types: Prop types are used for type checking the props a component receives to ensure that they conform to the expected data types.
  11. setState: The `setState` method is used to update the state of a component. It schedules a re-render with the uppdated state.
  12. componentDidMount: A lifestyle method called after a component is mounted to the DOM. It's commonly used for data fetching and setup.
  13. componentDidUpdate: A lifestyle method called after a component's state or props change. Itt's used for responding to changes.
  14. componentWillUnmount: A lifecycle method called before a component is removed from the DOM. It's used for cleanup and resource disposal.
  15. Key: A "key" is a special string attribute that needs to be added to elements inside an array to give the elements a stable identity. Keys help React identify which items have changed, been added, or removed.
  16. Event Handler: An event handler is a function that is called when a specific event occurs, such as a click event, and is used to respond to user interactions.
  17. Style-components: A popular CSS-in-JS library for styling React components by writing actual CSS code within JavaScript.
  18. React Router: A popular library for handling routing and navigation in React applications.
  19. useState: A hook used to add state to functional components.
  20. useEffect: A hook used to perform sode effects in functional components.
  21. useContext: A hook used for consuming context in functional components.
  22. Redux: A state management library for JavaScript applications, often use with React.
  23. Jest: A popular JavaScript testing framework often used for testing React applications.
  24. Enzyme: A JavaScript testing utility for React that makes it easier to assert, manipulate, and traverse React components' output.
  25. Deployment: The process of making a React application available on the internet for users to access.

Installation

To start using React in your project, you need to install it. You can install React via npm (Node Package Manager) or yarn. 

          
            # Using npm
            npm install react react-dom

            # Using yarn
            yarn add react react-dom

Hello World Example

To get you started, here's a simple "Hello World" example of a React component. 

          
            import React from 'react';

            function HelloWorld() {
             return <h1>Hello, World!</h1>;
            }

            export default HelloWorld;

Components

In React, applications are typically built by composing reusable components. Components can be functional or class-based, and they encapsulate a specific piece of functionality or user interface.

Functional Components

Functional components are JavaScript funtions that return JSX elements. They are simple and used for rendering UI elements. 

          
            function MyComponent(props) {
             return <div>{props.text}</div>
            }

Class Components

Class components are ES6 classes that extend the `React.Component` classs. They have a `render` method that returns JSX. 

          
            class MyComponent extends React.Component {
             render() {
              return <div>{this.props.text}</div>;
             }
            }

Props

Props are a way to passs data into a component. They are read-only, and the component recieving them cannot modify the props. Props are used to configure and customize a component. 

          
            function Greeting(props) {
             return <h1>Hello, (props.name)!</h1>;
            }

            // Usage
            <Greeting name="John" />
          
        
          
            import PropTypes from 'prop-types';

            function Greeting(props) {
             return <h1>Hello, {props.name}!</h1>;
            }

            Greeting.propTypes = {
             name: PropTypes.string.isRequired,
            };

State

State is used to store and manage data within a component. When state changes, the component re-renders o reflect the updated data. 

          
            class Counter extends React.Component {
             constructor(props) {
              super(props);
              this.state = {count: 0};
            }

             render() {
              return {
               <div>
                <p>Count: {this.state.count}</p>
                <button onClick={() => this.setState({ count: this.state.count + 1 })}>
                 Increment
                </button>
               </div>
              }
             }
            }

Lifecycle Methods

React components have various lifecycle methods that allow you to perform actions at specific points during a component's existence. Some common lifecycle methods include `componentDidMount`, `compoundDidUpdate`, and `compoundWillUnmount`.

Conditional Rendering

You can conditionally render elements based on certain conditions using JavaScript expressions. 

          
            function Greeting(props) {
               if (props.isLoggedIn) {
                return <h1>Welcome back!</h1>;
              } else {
                return <h1>Please sign up or log in.</h1>;
              }
            }

Event Handling

React allows you to define event handlers to respond to user interactions, such as cliks, form submissions, or keyboard input. 

          
            class Button extends React.Component {
             handleClick() {
              alert('Button clicked!');
             }

             render() {
              return <button onClick={this.handleClick}>Click Me</button>;
             }
            }

Forms

React provides a conveninet way to handle form inputs and manage their state. 

          
            class MyForm extends React.Component {
             constructor(props) {
              super(props);
              this.state = { inputValue: '' };
             }

             handleChange(event) {
              this.setState({ inputValue: event.target.value });
             }

             render() {
              return (
               <form>
                 input type="text" value={this.state.inputValue} onChange={(e) => this.handleChange(e)} />
                <p>You typed: {this.state.inputValue}</p>
               </form>
              );
             }
            }

Conditional Rendering 2

You can conditionally render components based on certain conditions. This can be achieved using JavaScript's conditional operators or conditional rendering functions. 

          
            function ConditionalComponent(props) {
             if (props.condition) {
              return <div>This is rendered when the condition is true.</div>;
             } else {
              return <div>This is rendered when the condition is false.</div>;
             }
            }

Router

For building single-page applications, React Router is commonly used to manage and navigate between different views or page within an application. 

          
            import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';

            function App() {
             return (
              <Router>
               <Switch>
                <Route exact path="/" component={Home} />
                <Route path="/about" component={About} />
                <Route path="/contact" component={Contact} />
               </Switch>
              </Router>
             );
            }

Hooks

Introduced in React 16.8, hooks allow yo to use state and ither React features in functional components. Commonly used hooks include `useState`, `useEffect`, and `useContext`. 

          
            import React, { useState, useEffect } from 'react';

            function Counter() {
             const [count, setCount] = useState(0);

             useEffect(() => {
              document.title = `Count: ${count}`;
             }, [count]);

             return (
              <div>
               <p>Count: {count}</p>
               <button onClick={() = > setCount(count + 1)}>Increment</button>
              </div>
             );
            }

Context

React Context allows data to be passed throgh the component tree without having to pass props manually at every level. it's often used for global state management. 

          
            const MyContext = React.createContext();

            function App() {
             return (
              <MyContext.Provider value="Hello from Context!">
               <MyComponent />
              </MyContext.Provider>
             );
            }

            function MyComponent() {
             const value = React.useContext(MyContext);
             return <p>{value}</p>;
            }
